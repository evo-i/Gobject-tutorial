#### Об этом учебнике

GObject является базовой системой для библиотеки GTK, текущая версия которой четвёртая.
GTK предоставляет графический интерфейс пользователя на Linux и используется системой рабочего стола GNOME и многими приложениями.
См. [учебник по GTK 4](https://github.com/ToshioCP/Gtk4-tutorial).
Одна из проблем в понимании GTK 4 заключается в сложности GObject.
Этот учебник полезен для тех, кто изучает GTK 4.
Читатели этого учебника также должны прочитать учебник по GTK 4, потому что GTK на данный момент является единственным применением GObject.

[Справочник API GObject](https://docs.gtk.org/gobject/) предлагает всё необходимое для GObject.
Содержание этого учебника не выходит за рамки документации.
Он просто показывает примеры и как писать программы на GObject.
Но я верю, что он полезен для начинающих, которые испытывают трудности в изучении системы GObject.
Читатели должны обращаться к документации GObject при изучении этого учебника.

#### Генерация GFM, HTML и PDF

Оглавление находится в конце этого файла, и вы можете просмотреть все учебные материалы по ссылке.
Однако вы можете создать GFM, HTML или PDF, выполнив следующие шаги.
GFM - это 'GitHub Flavored Markdown', который используется в файлах документации в репозитории GitHub.

1. Вам понадобятся операционная система Linux, ruby, rake, pandoc и система LaTeX.
2. скачайте [репозиторий GObject-tutorial](https://github.com/ToshioCP/Gobject-tutorial) и распакуйте файлы.
3. измените текущий каталог на верхний каталог файлов.
4. введите `rake` для создания файлов GFM. Файлы будут созданы в каталоге `gfm`.
5. введите `rake html` для создания файлов HTML. Файлы будут созданы в каталоге `docs`.
6. введите `rake pdf` для создания файла PDF. Файл будет создан в каталоге `latex`.

Эта система такая же, как и в репозитории `GTK 4 tutorial`.
В нём есть документ `Readme_for_developers.md` в каталоге `gfm`.
Он описывает детали.

#### Вклад в проект

Если у вас есть вопросы, не стесняйтесь создать issue.
Если вы найдёте какие-либо ошибки в учебнике, создайте issue или pull-request.
Когда вы отправляете pull-request, исправьте исходные файлы, которые находятся в каталоге 'src', и выполните `rake` и `rake html`.
Тогда файлы GFM и HTML будут автоматически обновлены.
