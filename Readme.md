#### О данном руководстве

GObject — это базовая система для библиотеки GTK, текущая версия которой — четыре.
GTK предоставляет графический интерфейс в Linux и используется в настольной системе GNOME и многих приложениях.
См. [руководство по GTK 4](https://github.com/ToshioCP/Gtk4-tutorial).
Одна из проблем в понимании GTK 4 — это сложность GObject.
Данное руководство полезно для тех, кто изучает GTK 4.
И читатели этого руководства должны также прочитать руководство по GTK 4, поскольку GTK — это единственное приложение GObject на данный момент.

[Справочник API GObject](https://docs.gtk.org/gobject/) содержит всё необходимое для работы с GObject.
Содержание данного руководства не выходит за рамки документации.
Оно лишь показывает примеры и как писать программы на GObject.
Но я полагаю, что оно полезно для новичков, которые испытывают трудности в изучении системы GObject.
Читатели должны обращаться к документации GObject при изучении данного руководства.

#### Генерация GFM, HTML и PDF

Содержание находится в конце данного файла, и вы можете увидеть все руководства по ссылкам.
Однако вы можете создать GFM, HTML или PDF, выполнив следующие шаги.
GFM — это 'GitHub Flavored Markdown', который используется в файлах документации в репозитории GitHub.

1. Вам потребуется операционная система Linux, ruby, rake, pandoc и система LaTeX.
2. скачайте [репозиторий GObject-tutorial](https://github.com/ToshioCP/Gobject-tutorial) и распакуйте файлы.
3. измените текущий каталог на верхний каталог файлов.
4. введите `rake` для создания файлов GFM. Файлы генерируются в каталоге `gfm`.
5. введите `rake html` для создания файлов HTML. Файлы генерируются в каталоге `docs`.
6. введите `rake pdf` для создания файла PDF. Файл генерируется в каталоге `latex`.

Эта система такая же, как и в репозитории `GTK 4 tutorial`.
В нём есть документ `Readme_for_developers.md` в каталоге `gfm`.
Он описывает детали.

#### Вклад

Если у вас есть какие-либо вопросы, не стесняйтесь создавать issue.
Если вы найдёте какие-либо ошибки в руководстве, создайте issue или pull-request.
Когда вы отправляете pull-request, исправьте исходные файлы, которые находятся в каталоге 'src', и запустите `rake` и `rake html`.
Тогда файлы GFM и HTML будут автоматически обновлены.
